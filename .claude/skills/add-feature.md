# Adding New Features Skill

When adding a new feature to TrialMine:
1. Plan first — explain what files will change before writing code
2. Add type hints and docstrings to all new public functions
3. Add Pydantic models for any new data structures
4. Write at least 2 unit tests for the new code
5. Update API schemas if the feature affects the API
6. Update CLAUDE.md "Current State" after completion
7. Commit with conventional commit message: feat:, fix:, docs:, refactor:
